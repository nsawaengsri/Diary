11-06-2018
ณ มหาวิทยาลัยเทคโนโลยีราชมงคล
วันนี้ฟังอาจารย์ทัศน์ พรีเซนต์เกี่ยวกับ ผลงานที่อาจารย์ได้ทำ พวก Robot , automatic 
ได้เห็นผลงานมากมาย
อาจารย์ก็เล่าให้ฟังว่าบทบาทของอาจารย์ก็จะทำการเข้าไปพูดคุยกับ User เพื่อเก็บ Req จากนั้นมาวิเคราะห์ หรือมองหา Sub จากคอนเนคชัน 
บอกแนวทาง ช่องทาง วิธีการ ทำในวงการธุรกิจนี้
อาจารย์มี Conneciton มี Project ให้เลือกทำ เริ่มจากโปรเจคเล็กๆ ก่อน
อาจารย์มีแหล่งซื้อ Robot 
อาจารย์เล่าการทำงานของการ Developer เกี่ยวกับผลงานกล้องจับภาพ ประมวลผม Image Processing ด้วย C
	เมื่อตัวกล้องตวจจับ QC ของ Item แล้วพบว่ามีบาโค้ด ไม่ถูกต้อง ก็จะทำการเก็บ LOG FILE TEXT ออกไปให้กับทาง
	เจ้าหน้าที่ฝ่าย IT เอาไปจัดการต่อเอง
	
อาจารย์เล่าว่า ส่วนใหญ่งานที่อาจารย์ได้รับจะเป็น Job นอก แล้วมักจะเป็นงานที่ต้องคิดสร้างสิ่งใหม่ๆ ที่ไม่เคยมีที่ไหนใช้มาก่อน
จากนั้นอาจารย์บอกว่า ถ้าทำงานจบแล้ว ลูกค้าส่วนใหญ่ ก็จะเอาผลงานที่เราสร้างคิดให้ ไปพัฒนาต่อยอด ดัดแปลง เอาเอง 
งานอาจารย์ ก็เลยมักจะจบเป็น Job ไปไม่ยืดยาวนับปี 

ส่วนเรื่องของคนทำงานก็ปัญหาอย่างหนึี่งเนื่องจากว่า โรงงานแต่ละที่ เมื่อมีการทำทีมที่จะต้องนำ Robot เข้ามาใช้
แล้วเมื่อบุคคลที่ทำหน้าที่ดูแล Robot ที่นำเข้ามาเกิดการย้ายออกไป ทำให้ทางโรงงาน ขาดผู้รับผิดขอบในส่วนนี้
สุดท้ายก็จะต้องลงทุนกับเจ้าใหม่เพื่อนำ Robot เข้ามาใช้งาน 

ส่วนทางเรา กำลังมองตนเองว่าจะหาช่องทางที่จะเริ่มจับทำธุรกิจนี้ได้อย่างไร
โดยมีความหวังว่าจะมีสักหนึ่งลูกค้า ที่กำลังต้องการนำ PLC มาใช้กับธุรกิจ จากนั้นก็ต่อยอดเข้ากับระบบ ERP ที่ทางเราถนัด


อาจารย์บอกว่า ถ้าหากว่า Hard Ware แป๊ะ อย่างอื่นก็สบาย
