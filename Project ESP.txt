 ไม่รู้
	- ต้องรู้จักตนก่อน ว่าปัจจุบัน อยู่ในจุดไหน
		งานในตอนนี้ จากอดีตมาจนถึงปัจจุบัน ทางเรามีโปรเจคการนำ LOT ใช้ในโรงงาน พร้อมทั้งเก็บค่าข้อมูลเพื่อนำมาประมวลผลต่อในระบบ NAV
จากนั้น ได้ทำการทดลองทำ LOT เพื่อส่งค่าข้อมูลระหว่างระบบ NAV 
จากการทดลอง ก็ทำให้ได้รู้ว่า สิ่งที่ต้องรู้ 1 เรื่องของ LOT แบบไหน ที่เขาใช้งานกัน เขาใช้งานกันอย่างไร เขาทำอย่างไรเพื่อให้อุปกรณ์ต่างๆ นั้นสามารถ สือสารกันได้
	1.1 เริ่มต้นที่จะต้องมีความรู้ เริ่มจาก การใช้อุปกรณ์เล็ก ที่จำลอง LOT คือ esp8266 จากนั้น ต้องรู้ว่า Programming ภาษาที่ใช้ เขียนอย่างไร จากนั้นต้องรุ้ว่า จะเขียนอย่างไร
ให้ตัว lot สามารถเชื่อมต่อกับ internet ได้
	1.2 เริ่มรู้แล้ว ก็มาทำลองกัน กับ nav ทางด้าน NAV ต้องหาวิธีการสื่อสารกับ ESP ต้องอาศัยความรู้เรื่องของ WEB SERVICE ว่าจะใช้แบบใด ในการเชื่อมต่อ
		1.2.1 ศึกษาเรื่องของ Web Service
		1.2.1 ศึกษาว่า NAV อนุญาติให้ใช้ Web Service ได้แบบใด มีทั้ง ODATA, SOAP 
		1.2.2 ศึกษาว่า ต้องเขียน Service ในรูปแบบใด เพื่อเรียก Webservice พบว่า ต้องเขียน เป็น JSON 
		1.2.3 ศึกษาหาวิธีของ NAV ที่เขียนเป็น JSON ในการเรียกเว็บ Service 
		1.2.4 ได้ทดลองทำการเขียน พบว่า สามารถส่งค่าไปร้องขอ Webservice ได้ 
		1.2.5 ทดลองต่อ โดยการเขียน ให้ไปเรียก Web service ของ ESP สามารถส่งค่าไปให้กับ ESP ได้
		1.2.6 ทดลองให้ ESP ส่งค่าเพื่อไปอัพเดท ข้อมูลใน NAV 
		1.2.7 กลับมาศึกษา การที่จะให้ Thridparty มายุ่งเกี่ยวกับ NAVจะต้องทำอย่างไรบ้าง ต้องมาดูว่าเปิด Service ยังไงเพื่อเป็นช่องทางในการรับข้อมูลมาอัพเดท
		1.2.8 ทดลองสร้าง Table page สำหรับรับข้อมูลจาก ESP จากนั้น ไปทำการเปิด Webservice 
		1.2.9 ESP ทำการสร้าง API มา 1 ตัว สำหรับติดต่อไปยัง Web Service ของ NAV 
		1.3.0 ศึกษาว่า จะใช้ภาษาใดสร้าง API ในการส่งค่าข้อมูลไปยัง NAV พบตัวเองที่สอน เป็น PHP ที่ติดต่อไปยัง Webservice NAV 
		1.3.1 ทดลองเขียน ให้ ESP ส่งค่ากลับไป API จากนั้น  API ส่งค่าไปอัพเดท NAV
	1.3 เมื่อเราทดลอง ในส่วนย่อยต่างๆ ที่เป็นส่วนประกอบของการเดินทาง เราก็ทำการทดสอบ แบบ FULL FLOW 
		1.3 เริ่มจาก ส่งค่าจาก NAV ไปยัง ESP ESP ทำการประมวลผล ส่งค่ากลับมายัง API จากนั้น API ทำการส่งค่าไปยัง WEB SERVICE NAV เพื่ออัพเดทข้อมูล 

- เส้นทางที่อยู่ในปัจจุบัน เป็นเส้นทางที่จะไปต่อ หรือไม่ 
	หลังจากไปทบทวนจุดยืนที่ยืนอยู่ พบว่า ยังอยู่ในเส้นทาง แต่เราต้องทำการทดลอง Advance เพิ่มขึ้น เพิ่มเติมจากการ รับ Requrement จากโรงงานจริง ไปดูอุปกรณ์จาก โรงงานจริง
ไปดูว่า อุปกรณ์ PLC ของโรงงาน สามารถให้เราเชื่อมต่อ เพื่อดึงข้อมูลมา ผ่านการสื่อสาร แบบใด ส่วนนี้ เราจะต้องอาศัยความรู้เรื่องการ Communicate ของ PLC 
ว่ามีแบบใด พบว่า ที่นิยมและเชื่อมต่อกับ คอมพิวเตอร์ได้ จะเป็น Port สื้อสารแบบ 232 แต่ตัว PLC เองเป็นแบบ 485 ตัว Flow mitor ตัวเครื่องชั่งเป็น 232 
จากนั้น ต้องมาดูว่า จะเชื่อมต่อต้องใช้อุปกรณ์ใด ต้องไปหาซื้อ ตัวเชื่อมต่อตัวแปลง PORT เพื่อใช้ต่อกับ ESP 
	จากนั้น สั่งซื้ออุปกรณ์ จำลอง FLOw งานจากการรับ Requriement ที่ทำให้ได้ครบ FLOW ก่อน 
จากการทดลองทำด้านบนมาทั้งหมด เราก็จะทดลอง จริงก่อน จากนั้น ก็ไปติดตั้งจริงให้กับลูกค้า
จุดอ่อนก็มีบ้าง เพราะนี่เป็นครั้งแรก ที่พวกเราทำกัน จึงหาวิธี ที่พวกเราสามารถทำกันได้ เพื่อสร้างสิ่งเหล่านี้ เพื่อต่อยอดในอนาคต หากพวกเราพบ ทางที่ดีกว่า ก็จะพัฒนาต่อไป
